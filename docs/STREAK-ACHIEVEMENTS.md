# استریک روزانه و اچیومنت — پیاده‌سازی (فاز ۲)

> قلاب‌های بازگشت (retention) + منبعِ اصلیِ ارزِ ⚡ کارت.
> کد: `backend/src/progress/`.

## ۱) استریکِ روزانه

هر روز که بازیکن جایزهٔ روزانه را «claim» کند، استریکش جلو می‌رود.

- «روز» بر مبنای **وقتِ تهران** (`Asia/Tehran`) و رشتهٔ `YYYY-MM-DD` محاسبه می‌شود.
- منطق (`streak.service.ts`):
  - اگر آخرین claim = **دیروز** → استریک +۱.
  - اگر = **امروز** → دوباره نمی‌شود (خطا).
  - در غیر این صورت (فاصله افتاده / اولین بار) → استریک = ۱.
- `streakCurrent`, `streakLongest`, `streakLastDay` روی `User` ذخیره می‌شوند.

### نردبانِ جایزه (`streak.constants.ts`, چرخه‌ایِ ۷ روزه)
| روز | جایزه |
|---|---|
| ۱ | ۲۰ 🪙 |
| ۲ | ۳۰ 🪙 |
| ۳ | ۴۰ 🪙 |
| ۴ | ۵۰ 🪙 |
| ۵ | ۶۰ 🪙 |
| ۶ | ۸۰ 🪙 |
| ۷ | ۱۰۰ 🪙 + ۱ ⚡ |

روزِ ۸ به بعد دوباره از الگوی روز ۱ شروع می‌شود (`(day-1) % 7`).

### API (پشتِ JWT)
| متد | مسیر | کار |
|---|---|---|
| GET | `/streak` | وضعیت: `current`, `longest`, `canClaimToday`, `claimableDay`, `claimableReward` |
| POST | `/streak/claim` | گرفتنِ جایزهٔ امروز → جایزه + اچیومنت‌های تازه‌باز‌شده |

## ۲) اچیومنت‌ها

تعریف‌ها در **کد** (`achievements.constants.ts`)؛ «باز شدن» در جدولِ
`user_achievements` ثبت می‌شود (یکتا per `userId,key`).

| کلید | شرط | جایزه |
|---|---|---|
| `first_win` | ۱ بردِ دوئل | ۱ ⚡ |
| `correct_50` | ۵۰ پاسخِ درست | ۱ ⚡ + ۵۰ 🪙 |
| `correct_100` | ۱۰۰ پاسخِ درست | ۲ ⚡ + ۱۰۰ 🪙 |
| `duel_10` | ۱۰ دوئل | ۱ ⚡ |
| `streak_7` | استریکِ ۷ روزه | ۲ ⚡ |

- متریک‌ها (`totalCorrect`, `duelsPlayed`, `duelWins`, `bestStreak`) از دادهٔ
  موجود محاسبه می‌شوند (بدونِ شمارندهٔ جدا) تا منبعِ حقیقت یکی بماند.
- `check(userId)` بعد از **پایانِ دوئل**، **پایانِ بازی سریع** و **claimِ استریک**
  صدا زده می‌شود؛ اچیومنت‌های واجدِ شرط را باز و جایزه‌شان را واریز می‌کند.

### API
| متد | مسیر | کار |
|---|---|---|
| GET | `/achievements` | فهرست با `unlocked`, `unlockedAt`, `progress`/`threshold` |

## ۳) تستِ دستی

```bash
curl localhost:3000/streak -H "Authorization: Bearer <T>"
curl -X POST localhost:3000/streak/claim -H "Authorization: Bearer <T>"
curl localhost:3000/achievements -H "Authorization: Bearer <T>"
```
رفتارِ تأییدشده: claimِ روز۱ = ۲۰ سکه، claimِ دومِ همان‌روز رد می‌شود، و «اولین برد»
پس از بردِ دوئل باز شده و ۱ کارت می‌دهد.

## ۴) کارهای بعدی
- **UI موبایل**: کارتِ استریک روی خانه + صفحهٔ اچیومنت.
- **مصرفِ کارت**: سوپرپاورها (VAR، نصف‌نصف…) — قدمِ بعدیِ فاز ۲.
- **اچیومنت‌های بیشتر** (هت‌تریک، فاتحِ دسته و…) با اضافه‌کردن به `ACHIEVEMENTS`.
- **پوشِ یادآوریِ استریک** (Pushe) که «استریکت نشکند».
