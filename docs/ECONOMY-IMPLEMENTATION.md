# اقتصاد — پیاده‌سازی (فاز ۲)

> این سند «چطور پیاده شد» است. طراحی و اعداد در `ECONOMY.md`.
> کد: `backend/src/economy/`.

## ۱) ارزها و جان

| ارز | فیلدِ User | منبع | خرج |
|---|---|---|---|
| 🪙 سکه | `coins` | اتمامِ بازی سریع، هر جوابِ درست، بردِ دوئل | پرکردنِ جان (بعداً: فروشگاه) |
| 🎽 هوادار | `fans` | بردِ/باختِ دوئل | (فاز بعد: صعودِ دسته) |
| ⚡ کارت | `cards` | (فاز بعد: استریک/اچیومنت) | (فاز بعد: سوپرپاور) |
| ❤️ جان | `lives` + `lastLifeRegenAt` | بازیابیِ خودکار، پرکردن با سکه | شروعِ هر بازی |

اعداد در `economy.constants.ts` (آینهٔ `ECONOMY.md`) — تیمِ بالانس همان‌جا تنظیم کند.

## ۲) بازیابیِ جان (سرور-محور)

- جان‌ها با گذرِ زمان پر می‌شوند: **یک جان هر `regenMinutes` دقیقه** (پیش‌فرض ۲۰).
- مبنا `lastLifeRegenAt` است. `applyRegen()` هنگام هر خواندنِ کیف‌پول یا خرج،
  تعدادِ جان‌های پرشده را حساب و ذخیره می‌کند (باقی‌ماندهٔ زمان حفظ می‌شود، نه ریست).
- `getWallet` مقدارِ `nextLifeInSec` را برمی‌گرداند تا کلاینت شمارشِ معکوس نشان دهد.

## ۳) دفترِ تراکنش (audit)

هر تغییرِ ارز یک ردیفِ `WalletTransaction` می‌سازد
(`currency`, `amount`±, `reason`). دلایلِ رایج: `duel_win`, `duel_loss`,
`duel_draw`, `duel_start`, `quick_start`, `quick_finish`, `refill_lives`.
این برای رفعِ اختلاف و آنالیزِ اقتصاد است.

## ۴) API (پشتِ JWT)

| متد | مسیر | کار |
|---|---|---|
| GET | `/economy/wallet` | کیف‌پول با جانِ محاسبه‌شده + `nextLifeInSec` |
| POST | `/economy/refill-lives` | پرکردنِ جان با `refillCost` سکه |

## ۵) اتصال به حلقهٔ بازی

- **دوئل** (`duel.service.ts`):
  - `find` → `spendLife(userId, 'duel_start')` (هر دوئل یک جان).
  - `maybeFinalize` → به هر بازیکنِ **انسانی**: `fans` (برد=`winDuel`، باخت/مساوی=
    `loseDuel`) + `coins = correct × perCorrect`. ربات‌ها جایزه نمی‌گیرند.
- **بازی سریع/بمب** (`match.service.ts`):
  - `createMatch` → خرجِ جان (فقط کاربرِ واقعی).
  - `finishMatch` → `coins = winQuick + correct × perCorrect` (فقط کاربرِ واقعی).

> «کاربرِ واقعی» یعنی نه ربات و نه مهمان (`isRealUser`). مهمان‌ها (فاز۰ بدون Auth)
> از اقتصاد معاف‌اند تا حلقهٔ تک‌نفرهٔ بدونِ ورود نشکند.

## ۶) تستِ دستی (خلاصه)

```bash
# کیف‌پول
curl localhost:3000/economy/wallet -H "Authorization: Bearer <TOKEN>"
# پرکردن جان
curl -X POST localhost:3000/economy/refill-lives -H "Authorization: Bearer <TOKEN>"
```
رفتارِ تأییدشده: خرجِ جان در هر بازی، خطای «جان کافی نداری» در صفر، پاداشِ
هوادار/سکه پس از دوئل، و کسرِ ۵۰ سکه هنگام پرکردنِ جان.

## ۷) کارهای بعدی

- **فروشگاه + Poolakey**: بستهٔ سکه/کارت، حذفِ تبلیغ، Battle Pass (اصل: pay-to-skip).
- **کارت/سوپرپاور**: خرجِ `cards` برای VAR، نصف‌نصف، وقت اضافه…
- **استریک روزانه + اچیومنت**: منبعِ اصلیِ `cards`.
- **باشگاه/صعود**: خرجِ سکه برای ارتقا، خرجِ هوادار برای صعودِ دسته
  (`promotion.fansNeeded`, `rewardMultiplier`).
- **بازیابیِ زندهٔ تایمرِ جان** با BullMQ (به‌جای محاسبهٔ lazy) اگر لازم شد.
