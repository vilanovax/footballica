# دستورالعمل ایجنت — فوتبالیکا (CLAUDE.md / .cursorrules)

> این فایل را در ریشهٔ ریپو بگذار. Claude Code آن را به‌صورت `CLAUDE.md`
> می‌خواند و Cursor به‌صورت `.cursorrules`. هر دو نام را می‌توانی کپی کنی.

## پروژه چیست
فوتبالیکا یک **بازیِ کوییز فوتبالی موبایل** است (نه اپ آزمون). RTL، فارسی،
بازار هدف ایران. مودها: بازی سریع، بمب، دوئل async، پنالتی، تورنمنت.

## استک (تغییر نده مگر با تأیید صریح)
- موبایل: React Native + Expo + expo-router + TypeScript
- state: Zustand + TanStack Query
- انیمیشن: Reanimated + Moti؛ آماده‌ها: Rive/Lottie؛ svg: react-native-svg
- بک‌اند: NestJS + PostgreSQL (Prisma) + Redis (ioredis) + BullMQ
- ادمین: Next.js

## قوانینِ غیرقابل‌نقض
1. **Server-authoritative:** کلاینت هرگز جوابِ درست یا deadline را تعیین نمی‌کند.
   سؤال بدون فیلد جواب به کلاینت می‌رود. درستی و زمان فقط سمت سرور سنجیده می‌شود.
   (به `match-engine.service.ts` نگاه کن — منبع حقیقت همان است.)
2. **async-first:** دوئل نوبتی است. تا فاز ۳ از Socket.IO استفاده نکن.
3. **همه‌چیز RTL و فارسی:** اعداد را با util فارسی نمایش بده (`toFa`)، تاریخ جلالی.
4. **ایران‌محور:** به‌جای Google Play/FCM/S3 → کافه‌بازار(Poolakey)/Pushe/آروان.
   هیچ وابستگی‌ای به سرویس تحریمی اضافه نکن بدون پرسیدن.

## منبع دیزاین
- توکن‌ها فقط از `src/theme.ts` می‌آیند. رنگ/فاصله/شعاعِ هاردکد ممنوع.
- ماک‌آپ‌های مرجع (HTML) در پوشهٔ `/design-mockups` هستند؛ ظاهر باید با آن‌ها بخواند:
  چمنِ راه‌راه، کهربایی فقط برای اکشن اصلی، دکمه‌های چرمیِ برجسته.

## وقتی کد می‌نویسی
- TypeScript سخت‌گیرانه؛ از `any` پرهیز کن.
- منطق بازی در بک‌اند، نه در کامپوننت‌های RN.
- هر endpoint جدید: ورودی را با DTO + class-validator اعتبارسنجی کن.
- قبل از افزودن کتابخانهٔ جدید، اول بپرس.
